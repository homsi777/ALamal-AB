# تقرير مفصل عن قسم التقارير وآلية عمله الحالية

## 1. مقدمة التقرير

هذا التقرير يشرح قسم **التقارير** في مشروع `ALamal-AB` كما هو مبني حالياً بعد تجهيز الأقسام الفرعية ومهام التقارير بشاشات مستقلة.

الهدف من التقرير هو توثيق:

- بنية قسم التقارير.
- آلية التنقل بين الأقسام والمهام.
- محتوى كل قسم فرعي.
- محتوى كل مهمة تقرير.
- الأزرار الموجودة حالياً في صفحة القسم وصفحة المهمة.
- نقاط القوة الحالية.
- المشكلة التصميمية الحالية: لماذا تبدو مهام التقارير شبيهة بباقي الشاشات وليست بتجربة تقارير حقيقية.
- توصيات تطوير لاحقة تجعل قسم التقارير مناسباً لمشروع ERP كبير.

## 2. الفكرة العامة لقسم التقارير

قسم التقارير ليس شاشة واحدة بسيطة، بل هو مركز تقارير كامل داخل النظام.

المفروض أن يخدم هذا القسم عدة أنواع من المستخدمين:

- المدير المالي.
- المحاسب العام.
- محاسب الذمم.
- محاسب الضرائب.
- محاسب التكاليف.
- الإدارة العليا.
- مسؤول الخزينة.
- مسؤول الأصول.

لذلك يجب أن تكون تجربة التقارير مختلفة عن تجربة الإدخال اليومي أو صفحات القوائم العادية.

التقارير عادة تحتاج:

- اختيار فترة.
- اختيار حساب أو عميل أو مورد أو مركز تكلفة.
- فلاتر متعددة.
- مقارنة بين فترات.
- عرض مجاميع.
- عرض تفاصيل.
- طباعة.
- تصدير Excel أو PDF.
- حفظ إعدادات التقرير.
- أحياناً جدولة إرسال التقرير.

حالياً تم تجهيز الهيكل الأساسي، لكن تصميم شاشة المهمة لا يزال عاماً ويشبه باقي صفحات النظام أكثر مما يشبه شاشة تقارير احترافية.

## 3. الملفات المرتبطة بقسم التقارير

### 3.1 ملف الراوتر الرئيسي للتقارير

الملف:

```text
src/pages/ReportsPage.tsx
```

وظيفته:

- تعريف مسارات قسم التقارير.
- تحويل `/reports` إلى `/reports/financial-statements`.
- فتح صفحة قسم فرعي عند المسار:

```text
/reports/:sectionId
```

- فتح شاشة مهمة مستقلة عند المسار:

```text
/reports/:sectionId/:moduleId
```

أمثلة:

```text
/reports/accounts
/reports/accounts/customer-statement
/reports/financial-statements/balance-sheet
/reports/management-bi/dashboard
```

### 3.2 ملف صفحة القسم الفرعي

الملف:

```text
src/pages/reports/ReportsSectionPage.tsx
```

وظيفته:

- عرض عنوان القسم الفرعي.
- عرض وصف القسم الفرعي.
- عرض أزرار عامة أعلى الصفحة.
- عرض مؤشرات مختصرة للقسم.
- عرض بطاقات مهام التقارير.
- عند الضغط على بطاقة مهمة يتم الانتقال إلى شاشة مستقلة للمهمة.

### 3.3 ملف صفحة مهمة التقرير المستقلة

الملف:

```text
src/pages/reports/ReportsModulePage.tsx
```

وظيفته:

- قراءة `sectionId` و `moduleId` من الرابط.
- التحقق من صحة القسم.
- البحث عن المهمة داخل بيانات التقارير.
- إذا كانت المهمة غير موجودة يرجع المستخدم إلى القسم.
- عرض عنوان المهمة.
- عرض وصف المهمة.
- عرض زر رجوع.
- عرض زر الإجراء الرئيسي للمهمة.
- عرض زر التصدير.
- عرض مؤشرات القسم.
- عرض مسار عمل التقرير.
- عرض جدول بيانات أولي.
- عرض نسخة موبايل من بيانات الجدول.

### 3.4 ملف تعريف الأقسام والبطاقات

الملف:

```text
src/data/reportsSections.ts
```

وظيفته:

- تعريف الأقسام الفرعية.
- تعريف أسماء مهام كل قسم.
- تعريف الأيقونة والوصف لكل بطاقة.
- دعم الأقسام البسيطة والأقسام التي تحتوي مجموعات.

مثال على قسم بسيط:

```text
financial-statements
```

مثال على قسم يحتوي مجموعات:

```text
accounts
```

### 3.5 ملف بيانات شاشة التقارير

الملف:

```text
src/data/reportsWorkspace.ts
```

وظيفته:

- تعريف مؤشرات كل قسم.
- تعريف مسار عمل كل قسم.
- تعريف بيانات كل مهمة تقرير.
- تعريف الإجراء الرئيسي لكل مهمة.
- توليد معرف المهمة من مفتاح الترجمة.
- البحث عن مهمة تقرير حسب القسم والمعرف.

## 4. آلية التنقل الحالية

### 4.1 الدخول إلى قسم التقارير

عند فتح:

```text
/reports
```

يتم تحويل المستخدم تلقائياً إلى:

```text
/reports/financial-statements
```

هذا يعني أن قسم **القوائم المالية** هو القسم الافتراضي.

### 4.2 فتح قسم فرعي

عند فتح:

```text
/reports/accounts
```

تظهر صفحة قسم **تقارير الحسابات**.

الصفحة تعرض:

- عنوان القسم.
- وصف القسم.
- أزرار عامة.
- مؤشرات مختصرة.
- مجموعات المهام إن وجدت.
- بطاقات المهام.

### 4.3 فتح مهمة تقرير

عند الضغط على بطاقة مثل **كشف حساب عميل** يتم الانتقال إلى:

```text
/reports/accounts/customer-statement
```

وهنا تظهر شاشة مستقلة للمهمة.

هذا السلوك أفضل من عرض المحتوى أسفل الصفحة، لأنه:

- يمنع النزول الطويل داخل صفحة واحدة.
- يعطي المهمة مساحة عمل خاصة.
- يناسب مشروع كبير.
- يسمح لاحقاً بإضافة فلاتر وتبويبات وتصدير خاص بكل تقرير.
- يجعل الرابط قابلاً للمشاركة أو الحفظ.

## 5. الأزرار الحالية في صفحة القسم الفرعي

في صفحة القسم مثل:

```text
/reports/accounts
```

الأزرار الموجودة أعلى الصفحة حالياً:

| الزر | مكانه | وظيفته الحالية | الملاحظة |
|---|---|---|---|
| تصدير | أعلى الصفحة داخل `PageHeader` | زر واجهة فقط حالياً | يحتاج تحديد هل يصدر كل تقارير القسم أم تقرير مختار |
| قريباً | أعلى الصفحة داخل `PageHeader` | زر واجهة فقط | وجوده مؤقت وغير مناسب على المدى الطويل |

ملاحظة مهمة:

زر **تصدير** في صفحة القسم غير واضح وظيفياً، لأن القسم يحتوي عدة تقارير. الأفضل مستقبلاً إما:

- إزالته من صفحة القسم.
- أو جعله يفتح قائمة خيارات.
- أو جعله يصدر ملخص القسم فقط.
- أو استبداله بزر "إعدادات التقارير" أو "تقرير جديد".

## 6. الأزرار الحالية في شاشة مهمة التقرير

في شاشة مهمة مثل:

```text
/reports/accounts/customer-statement
```

الأزرار الموجودة حالياً:

| الزر | مكانه | وظيفته الحالية | الملاحظة |
|---|---|---|---|
| رجوع | أعلى الصفحة داخل `PageHeader` | يرجع إلى صفحة القسم الفرعي | زر مهم وصحيح لتجربة الشاشة المستقلة |
| زر الإجراء الرئيسي | أعلى الصفحة | يختلف حسب المهمة، مثل "كشف عميل" أو "فتح الميزانية" | حالياً زر واجهة فقط ويحتاج لاحقاً إلى تنفيذ فعلي |
| تصدير | أعلى الصفحة | زر واجهة فقط | يجب لاحقاً تحديد PDF / Excel / طباعة |

الأزرار الحالية جيدة كبداية، لكنها غير كافية لتجربة تقارير حقيقية.

الأزرار المتوقعة مستقبلاً في كل مهمة:

- رجوع.
- تحديث التقرير.
- تطبيق الفلاتر.
- مسح الفلاتر.
- تصدير PDF.
- تصدير Excel.
- طباعة.
- حفظ إعدادات التقرير.
- جدولة الإرسال.
- مشاركة واتساب أو بريد إن كان مطلوباً.

## 7. الأقسام الفرعية الموجودة في التقارير

قسم التقارير يحتوي حالياً 8 أقسام فرعية:

| القسم | المسار | طبيعة القسم |
|---|---|---|
| القوائم المالية | `/reports/financial-statements` | تقارير مالية رسمية |
| تقارير الحسابات | `/reports/accounts` | حسابات، ذمم، كشوف |
| الموازنة والتحليل | `/reports/budget-analysis` | مقارنة فعلي ومخطط |
| الخزينة والبنوك | `/reports/treasury` | نقد، بنوك، تدفق |
| الأصول الثابتة | `/reports/fixed-assets` | أصول، إهلاك، استبعاد |
| الضرائب | `/reports/tax` | VAT، إقرارات، امتثال |
| التكاليف | `/reports/cost` | COGS، هامش، نقطة تعادل |
| BI والإدارة | `/reports/management-bi` | مؤشرات ولوحات تنفيذية |

## 8. تفاصيل قسم القوائم المالية

المسار:

```text
/reports/financial-statements
```

طبيعة القسم:

هذا القسم مخصص للتقارير المالية الرسمية التي تعتمد عليها الإدارة والمحاسبة في تقييم وضع الشركة.

المؤشرات الحالية في القسم:

- قوائم جاهزة.
- صافي الربح.
- أصول متداولة.
- ملاحظات تدقيق.

مسار العمل الحالي:

1. تجميع الأرصدة.
2. تطبيق التسويات.
3. مراجعة الإدارة.
4. تصدير التقرير.

المهام الموجودة:

| المهمة | المسار | زر الإجراء الرئيسي | وصف المهمة |
|---|---|---|---|
| الميزانية العمومية | `/reports/financial-statements/balance-sheet` | فتح الميزانية | عرض الأصول والخصوم وحقوق الملكية |
| قائمة الدخل | `/reports/financial-statements/income-statement` | فتح قائمة الدخل | عرض الإيرادات والتكلفة والمصاريف وصافي الربح |
| التدفقات النقدية | `/reports/financial-statements/cash-flow` | فتح التدفقات | عرض التدفقات التشغيلية والاستثمارية والتمويلية |
| تغيرات حقوق الملكية | `/reports/financial-statements/equity-changes` | فتح حقوق الملكية | عرض تغيرات حقوق الملكية خلال الفترة |

ما يجب أن تحتويه هذه الشاشات مستقبلاً:

- اختيار الفترة المالية.
- مقارنة مع فترة سابقة.
- عرض أرقام إجمالية أعلى التقرير.
- تقسيم واضح حسب بنود القوائم.
- زر تصدير PDF رسمي.
- زر Excel.
- زر طباعة.
- توقيع أو اعتماد داخلي.

## 9. تفاصيل قسم تقارير الحسابات

المسار:

```text
/reports/accounts
```

طبيعة القسم:

هذا هو أكثر قسم تشغيلي داخل التقارير، لأنه يخدم الحسابات العامة والذمم المدينة والدائنة.

المؤشرات الحالية:

- حسابات نشطة.
- كشوف جاهزة.
- أرصدة متأخرة.
- فروقات مراجعة.

مسار العمل الحالي:

1. اختيار الحساب.
2. تحديد الفترة.
3. مراجعة الحركات.
4. طباعة أو تصدير.

هذا القسم مقسم إلى ثلاث مجموعات:

- الأستاذ العام.
- الذمم المدينة.
- الذمم الدائنة.

### 9.1 مجموعة الأستاذ العام

| المهمة | المسار | زر الإجراء الرئيسي | وصف المهمة |
|---|---|---|---|
| كشف حساب لكل حساب | `/reports/accounts/account-statement` | فتح كشف حساب | يعرض حركة حساب محدد مع الرصيد الافتتاحي والختامي |
| تفصيل قيود اليومية | `/reports/accounts/journal-detail` | عرض القيود | يعرض القيود حسب الحساب والمستخدم ومركز التكلفة |
| ميزان المراجعة | `/reports/accounts/trial-balance` | عرض الميزان | يعرض إجمالي المدين والدائن وفروقات الميزان |

### 9.2 مجموعة الذمم المدينة

| المهمة | المسار | زر الإجراء الرئيسي | وصف المهمة |
|---|---|---|---|
| أعمار الديون المدينة | `/reports/accounts/ar-aging` | فتح أعمار العملاء | يعرض الذمم حسب فترات التأخير |
| كشف حساب عميل | `/reports/accounts/customer-statement` | كشف عميل | يعرض كشف العميل للطباعة والإرسال |
| مخاطر التأخير | `/reports/accounts/overdue-risk` | تحليل المخاطر | يحلل العملاء عاليي المخاطر والتأخير |

### 9.3 مجموعة الذمم الدائنة

| المهمة | المسار | زر الإجراء الرئيسي | وصف المهمة |
|---|---|---|---|
| أعمار الذمم الدائنة | `/reports/accounts/ap-aging` | فتح أعمار الموردين | يعرض الذمم حسب المورد والاستحقاق |
| كشف حساب مورد | `/reports/accounts/supplier-statement` | كشف مورد | يعرض فواتير ومدفوعات المورد |
| المدفوعات المستحقة | `/reports/accounts/payment-due` | جدول الدفع | يعرض جدول المدفوعات حسب الأولوية والسيولة |

ملاحظة تصميمية:

هذا القسم تحديداً لا يجب أن يبدو كجدول عام فقط. يجب أن يحتوي كل تقرير على فلاتر أعلى الشاشة:

- الحساب.
- العميل.
- المورد.
- من تاريخ.
- إلى تاريخ.
- العملة.
- الحالة.
- تجميع حسب.

## 10. تفاصيل قسم الموازنة والتحليل

المسار:

```text
/reports/budget-analysis
```

طبيعة القسم:

هذا القسم مخصص للمقارنة بين المخطط والفعلي وتحليل الانحرافات.

المؤشرات الحالية:

- نسبة استخدام الموازنة.
- انحراف إيرادات.
- انحراف مصاريف.
- مراكز تكلفة.

مسار العمل الحالي:

1. تحميل الموازنة.
2. جلب الفعلي.
3. احتساب الانحراف.
4. توصية إجراء.

المهام:

| المهمة | المسار | زر الإجراء الرئيسي | وصف المهمة |
|---|---|---|---|
| الفعلي مقابل المخطط | `/reports/budget-analysis/actual-vs-plan` | مقارنة الفعلي | مقارنة الفعلي بالمخطط حسب البند والفترة |
| تحليل الانحراف | `/reports/budget-analysis/variance` | تحليل الانحراف | تحليل أسباب الانحرافات |
| مراكز التكلفة | `/reports/budget-analysis/cost-centers` | مراكز التكلفة | قياس أداء مراكز التكلفة |
| الربحية | `/reports/budget-analysis/profitability` | عرض الربحية | ربحية الأقسام والمنتجات والعملاء |

ما ينقص تصميمياً:

- رسوم بيانية للفروقات.
- ألوان دلالية للانحرافات.
- فلاتر موازنة/فترة/مركز تكلفة.
- ملخص تنفيذي أعلى الشاشة.
- توصيات أو تعليقات إدارية.

مسار العمل الحالي:

3. تحليل التدفق.

## 12. تفاصيل قسم الأصول الثابتة

المسار:

```text
/reports/fixed-assets
```

طبيعة القسم:

مخصص لسجل الأصول والإهلاك والاستبعاد وإعادة التقييم.

المؤشرات الحالية:

- أصول مسجلة.
- القيمة الدفترية.
- إهلاك الفترة.
- أصول مستبعدة.

مسار العمل الحالي:

1. جرد الأصول.
2. احتساب الإهلاك.
3. مراجعة الاستبعاد.
4. تصدير السجل.

المهام:

| المهمة | المسار | زر الإجراء الرئيسي | وصف المهمة |
|---|---|---|---|
| سجل الأصول | `/reports/fixed-assets/register` | سجل الأصول | عرض الأصول حسب الموقع والمسؤول والقيمة |
| الإهلاك | `/reports/fixed-assets/depreciation` | تقرير الإهلاك | عرض الإهلاك الشهري والمتراكم |
| الأصول المستبعدة | `/reports/fixed-assets/disposed` | الأصول المستبعدة | عرض الأصول المستبعدة وربح أو خسارة التخلص |
| إعادة التقييم | `/reports/fixed-assets/revaluation` | إعادة التقييم | مقارنة القيمة العادلة بالدفترية |

نواقص تصميمية:

- لا يوجد فلاتر حسب موقع الأصل.
- لا يوجد فلاتر حسب نوع الأصل.
- لا يوجد عرض عمر الأصل.
- لا يوجد ملخص إهلاك واضح.
- لا يوجد مخطط حركة قيمة الأصل.

## 13. تفاصيل قسم الضرائب

المسار:

```text
/reports/tax
```

طبيعة القسم:

مخصص لضريبة VAT والإقرارات والامتثال والاقتطاعات.

## 16. مكونات شاشة مهمة التقرير الحالية

كل شاشة مهمة تحتوي حالياً على:

### 16.1 رأس الصفحة

يعرض:

- عنوان المهمة.
- اسم القسم + وصف المهمة.
- زر رجوع.
- زر الإجراء الرئيسي.
- زر تصدير.

### 16.2 مؤشرات القسم

تعرض نفس مؤشرات القسم الفرعي.

مثلاً داخل مهمة من قسم الحسابات، تظهر:

- حسابات نشطة.
- كشوف جاهزة.
- أرصدة متأخرة.
- فروقات مراجعة.

### 16.3 لوحة تفاصيل المهمة

تحتوي:

- عنوان الإجراء.
- وصف المهمة.
- مسار عمل من 4 خطوات.
- جدول بيانات.
- عرض موبايل للبيانات.

### 16.4 جدول البيانات

الأعمدة الحالية:

| العمود | وظيفته |
|---|---|
| المرجع | كود الصف |
| التقرير | اسم البند أو التقرير |
| الفترة | فترة التقرير |
| القيمة | قيمة مالية أو نسبة |
| المسؤول | الجهة أو الدور المسؤول |
| الحالة | جاهز، قيد المراجعة، تم التصدير، مجدول |

## 17. المشكلة التصميمية الحالية

رغم أن الهيكل صار أفضل من السابق، إلا أن شاشة مهمة التقرير لا تزال تشبه باقي شاشات النظام.

المشكلة ليست في الألوان فقط، بل في طبيعة التجربة:

- الشاشة لا تحتوي فلاتر تقرير واضحة.
- لا يوجد شريط إعدادات التقرير.
- لا يوجد اختيار فترة في مقدمة الشاشة.
- لا يوجد عرض نتيجة تقرير بصيغة محاسبية حقيقية.
- لا يوجد فرق بصري كاف بين تقرير مالي وتقرير خزينة وتقرير BI.
- الجداول عامة جداً.
- الأزرار عامة وليست مخصصة لطبيعة التقرير.
- لا توجد تجربة "إعداد ثم تشغيل التقرير".

بمعنى آخر:

التصميم الحالي جهز الهيكل، لكنه لم يصل بعد إلى تجربة "مهمة تقرير" حقيقية.

## 18. لماذا يجب أن تختلف مهام التقارير عن باقي الشاشات

صفحات النظام العادية عادة تكون:

- إدخال فاتورة.
- قائمة أصناف.
- قائمة عملاء.
- شاشة إعدادات.
- شاشة متابعة.

أما شاشة التقرير فهي مختلفة لأنها تعتمد على:

- شروط.
- فترة.
- تجميع.
- مقارنة.
- تصدير.
- نتيجة محسوبة.

لذلك يجب أن تكون شاشة التقرير مبنية حول السؤال:

**ماذا يريد المستخدم أن يستخرج؟**

وليس فقط:

**ما هي البيانات المعروضة؟**

## 19. التصميم المقترح لشاشة مهمة التقرير

يوصى بأن تصبح شاشة كل مهمة تقرير مكونة من مناطق واضحة:

### 19.1 شريط إعداد التقرير

يوجد أعلى الشاشة مباشرة بعد العنوان.

يحتوي:

- من تاريخ.
- إلى تاريخ.
- الحساب أو العميل أو المورد.
- العملة.
- الفرع.
- مركز التكلفة.
- حالة الفاتورة أو القيد.
- زر تطبيق.
- زر مسح.

### 19.2 شريط ملخص سريع

يعرض:

- إجمالي التقرير.
- عدد السجلات.
- أكبر قيمة.
- صافي الحركة.
- الرصيد الافتتاحي والختامي عند الحاجة.

### 19.3 منطقة النتيجة

تختلف حسب نوع التقرير:

- القوائم المالية: جداول محاسبية رسمية.
- كشف الحساب: رصيد افتتاحي، حركات، رصيد ختامي.
- أعمار الديون: buckets عمرية.
- الخزينة: تدفقات ومطابقات.
- BI: رسوم ومؤشرات.

### 19.4 شريط الإجراءات

يحتوي:

- تحديث.
- طباعة.
- PDF.
- Excel.
- حفظ الإعداد.
- جدولة.

## 20. تقييم الأزرار الحالية

### 20.1 زر رجوع

تقييمه:

جيد ومهم.

سبب أهميته:

- المستخدم ينتقل إلى شاشة مستقلة.
- يحتاج طريق رجوع واضح.
- يمنع الضياع داخل التقارير.

### 20.2 زر الإجراء الرئيسي

تقييمه:

جيد كمؤشر نية، لكنه غير كاف.

مثلاً:

- "كشف عميل" يجب أن يفتح أو يطبق التقرير بعد اختيار العميل.
- "فتح الميزانية" يجب أن يعرض ميزانية بفترة محددة.
- "تحليل الهامش" يجب أن يطبق فلاتر الهامش.

حالياً هو زر واجهة فقط.

### 20.3 زر تصدير

تقييمه:

ضروري، لكن يجب تفصيله.

الأفضل أن يتحول إلى:

- تصدير PDF.
- تصدير Excel.
- طباعة.

أو زر واحد يفتح قائمة خيارات.

### 20.4 زر قريباً

تقييمه:

غير مناسب لقسم تقارير في نظام كبير.

الأفضل:

- إزالته.
- أو استبداله بزر "إعدادات التقرير".
- أو "تقرير مخصص".

## 21. توصيات تنفيذية قصيرة المدى

هذه التوصيات يمكن تنفيذها بدون تغيير جذري:

1. إزالة زر "قريباً" من صفحات التقارير.
2. إضافة شريط فلاتر أعلى شاشة المهمة.
3. جعل زر الإجراء الرئيسي اسمه "تطبيق التقرير" أو "تحديث التقرير".
4. فصل زر التصدير إلى PDF و Excel.
5. إضافة حالة فارغة إذا لم يتم اختيار الفلاتر.
6. إضافة عنوان فرعي واضح يشرح الفترة الحالية.
7. تخصيص الأعمدة حسب نوع التقرير بدلاً من جدول موحد.

## 22. توصيات متوسطة المدى

1. إنشاء مكون مشترك باسم `ReportTaskShell`.
2. إنشاء مكون `ReportFilterBar`.
3. إنشاء مكون `ReportActionBar`.
4. إنشاء مكون `ReportSummaryStrip`.
5. إنشاء أنواع بيانات مختلفة لكل نوع تقرير.
6. ربط التقارير ببيانات حقيقية تدريجياً.
7. إضافة حفظ إعدادات التقرير في local state أو backend لاحقاً.
8. إضافة طباعة PDF حقيقية.

## 23. توصيات بعيدة المدى

1. بناء محرك تقارير داخلي.
2. دعم تقارير مخصصة.
3. دعم جدولة إرسال التقارير.
4. دعم صلاحيات حسب التقرير.
5. دعم اعتماد تقارير مالية.
6. دعم أرشفة النسخ المصدرة.
7. دعم مقارنة الفترات بصرياً.
8. دعم Dashboard حقيقي لقسم BI.

## 24. الخلاصة

قسم التقارير حالياً أصبح يمتلك هيكلاً جيداً:

- أقسام فرعية واضحة.
- مهام متعددة.
- روابط مستقلة لكل مهمة.
- زر رجوع.
- بيانات أمامية جاهزة.
- مسار عمل لكل قسم.
- جداول أولية.

لكن المشكلة الأساسية أن **تصميم شاشة مهمة التقرير لا يزال عاماً**.

لكي يصبح مناسباً لمشروع ERP كبير، يجب أن تتحول كل مهمة تقرير إلى تجربة خاصة بها:

- فلاتر واضحة.
- نتيجة تقرير حقيقية.
- أزرار تصدير وطباعة واضحة.
- عرض محاسبي أو إداري حسب نوع التقرير.
- اختلاف بصري ووظيفي بين تقرير وآخر.

الخطوة الصحيحة التالية ليست إضافة المزيد من البطاقات، بل تصميم قالب شاشة مهمة تقرير احترافي يستخدم لكل تقرير، مع تخصيص الفلاتر والنتائج حسب نوع المهمة.

# تقرير مفصل عن قسم المالية وآلية عمله الحالية

## 1. مقدمة التقرير

هذا التقرير يشرح قسم **المالية** في مشروع `ALamal-AB` كما هو مبني حالياً داخل الواجهة الأمامية.

الهدف من التقرير هو توثيق:

- بنية قسم المالية.
- آلية التنقل بين الأقسام المالية الفرعية.
- محتوى كل قسم فرعي.
- المهام أو الوحدات الموجودة داخل كل قسم.
- الأزرار الحالية في كل شاشة.
- المؤشرات الظاهرة في كل قسم.
- مسار العمل المعروض لكل قسم.
- طبيعة الجداول والبيانات التجريبية الحالية.
- نقاط القوة الحالية.
- الفجوات الوظيفية والتصميمية.
- توصيات تطوير لاحقة حتى يصبح قسم المالية Workspace مالي حقيقي وليس فقط واجهة عرض.

## 2. الفكرة العامة لقسم المالية

قسم المالية هو العمود التشغيلي والمحاسبي للنظام.

هذا القسم لا يجب أن يكون مجرد قائمة بطاقات أو جدول ثابت، بل يجب أن يتحول تدريجياً إلى مركز عمل مالي كامل يخدم:

- المحاسب العام.
- محاسب الذمم المدينة.
- محاسب الذمم الدائنة.
- مسؤول الخزينة.
- المدير المالي.
- محاسب الأصول.
- محاسب الضرائب.
- محاسب التكاليف.
- الإدارة العليا عند متابعة الوضع المالي.

حالياً تم تجهيز قسم المالية كواجهة أمامية منظمة، ويعرض:

- أقسام مالية فرعية.
- مؤشرات مختصرة لكل قسم.
- بطاقات مهام داخلية.
- لوحة تفاصيل عند اختيار مهمة.
- مسار عمل Workflow.
- جدول بيانات أولي.
- عرض موبايل بديل للجدول.

لكن القسم لا يزال **واجهة أمامية تشغيلية أولية**، وليس بعد نظاماً مالياً كاملاً متصل البيانات أو قابل للإدخال والاعتماد والتصدير الكامل.

## 3. الملفات المرتبطة بقسم المالية

### 3.1 ملف الراوتر الرئيسي للمالية

الملف:

```text
src/pages/FinancePage.tsx
```

وظيفته:

- تعريف مسارات قسم المالية.
- تحويل `/finance` إلى `/finance/general-ledger`.
- فتح صفحة القسم الفرعي حسب المسار.

المسارات الحالية:

```text
/finance/general-ledger
/finance/payable
/finance/receivable
/finance/treasury
/finance/fixed-assets
/finance/budgeting
/finance/tax
/finance/cost
```

ملاحظة:

كل مسار من هذه المسارات يفتح نفس المكون `FinanceSectionPage` مع تغيير `sectionId`.

### 3.2 ملف صفحة القسم المالي الفرعي

الملف:

```text
src/pages/finance/FinanceSectionPage.tsx
```

وظيفته:

- عرض عنوان القسم المالي.
- عرض وصف القسم.
- عرض زر الإجراء الرئيسي للمهمة المختارة حالياً.
- عرض مؤشرات القسم.
- عرض بطاقات المهام.
- السماح للمستخدم باختيار مهمة من داخل القسم.
- عرض لوحة تفاصيل للمهمة المختارة.
- عرض Workflow خاص بالقسم.
- عرض جدول بيانات للمهمة.
- عرض نسخة موبايل من الصفوف.

### 3.3 ملف تعريف الأقسام والبطاقات

الملف:

```text
src/data/financeSections.ts
```

وظيفته:

- تعريف الأقسام المالية الفرعية.
- تعريف الوحدات أو المهام داخل كل قسم.
- تعريف الأيقونة.
- تعريف مفاتيح الترجمة للعنوان والوصف.

### 3.4 ملف بيانات واجهة المالية

الملف:

```text
src/data/financeWorkspace.ts
```

وظيفته:

- تعريف مؤشرات كل قسم.
- تعريف Workflow لكل قسم.
- تعريف بيانات كل مهمة.
- تعريف زر الإجراء الرئيسي لكل مهمة.
- تعريف صفوف الجدول التجريبية.
- تعريف حالات الصفوف مثل: جاهز، مرحّل، قيد الاعتماد، مسودة، مطابق.

## 4. آلية التنقل الحالية

### 4.1 الدخول إلى قسم المالية

عند فتح:

```text
/finance
```

يتم تحويل المستخدم تلقائياً إلى:

```text
/finance/general-ledger
```

أي أن قسم **المحاسبة العامة** هو القسم الافتراضي للمالية.

### 4.2 فتح قسم مالي فرعي

مثال:

```text
/finance/treasury
```

يفتح قسم **الخزينة والنقدية**.

الصفحة تعرض:

- عنوان القسم.
- وصف القسم.
- زر إجراء رئيسي أعلى الصفحة.
- مؤشرات مختصرة.
- بطاقات المهام.
- لوحة تفاصيل المهمة المختارة.

### 4.3 اختيار مهمة داخل القسم

عند الضغط على بطاقة داخل القسم، لا يتم فتح صفحة مستقلة حالياً.

السلوك الحالي:

- يتم تغيير المهمة النشطة داخل نفس الصفحة.
- يتم تحديث زر الإجراء الرئيسي.
- يتم تحديث عنوان لوحة التفاصيل.
- يتم تحديث الوصف.
- يتم تحديث الجدول.

مثال:

داخل قسم المحاسبة العامة يمكن الضغط على:

- دفتر الأستاذ العام.
- قيود اليومية.
- ميزان المراجعة.
- القوائم المالية.

كل ضغطة تبدّل محتوى اللوحة أسفل البطاقات.

## 5. الأزرار الحالية في قسم المالية

### 5.1 زر الإجراء الرئيسي أعلى الصفحة

يوجد في `PageHeader`.

هذا الزر يتغير حسب المهمة المختارة.

أمثلة:

| القسم | المهمة | نص الزر |
|---|---|---|
| المحاسبة العامة | دفتر الأستاذ العام | فتح شجرة الحسابات |
| المحاسبة العامة | قيود اليومية | قيد يومية جديد |
| الحسابات الدائنة | فواتير الموردين | فاتورة مورد جديدة |
| الحسابات الدائنة | أوامر الدفع | إنشاء أمر دفع |
| الحسابات المدينة | التحصيلات | سند قبض جديد |
| الخزينة | المطابقة البنكية | مطابقة جديدة |
| الأصول الثابتة | سجل الأصول | إضافة أصل |
| الموازنة | إعداد الموازنة | إعداد موازنة |
| الضرائب | حساب الضريبة | احتساب الفترة |
| التكاليف | تكلفة الإنتاج | فتح تكلفة إنتاج |

وظيفته الحالية:

- زر واجهة فقط.
- لا يفتح نموذجاً فعلياً بعد.
- لا ينفذ عملية حفظ أو اعتماد.
- يوضح نية المهمة فقط.

### 5.2 بطاقات المهام

كل بطاقة تعمل كزر اختيار.

عند الضغط عليها:

- تصبح البطاقة فعالة بصرياً.
- تتغير لوحة التفاصيل.
- يتغير زر الإجراء الرئيسي.

هذا جيد كبداية، لكنه مختلف عن التقارير الجديدة التي أصبحت تفتح شاشة مستقلة لكل مهمة.

### 5.3 أزرار غير موجودة حالياً ولكن متوقعة مستقبلاً

قسم المالية يحتاج لاحقاً أزراراً متخصصة حسب نوع المهمة:

- جديد.
- تعديل.
- اعتماد.
- ترحيل.
- إلغاء اعتماد.
- عكس قيد.
- طباعة.
- تصدير PDF.
- تصدير Excel.
- مطابقة.
- إقفال فترة.
- احتساب.
- جدولة.
- أرشفة.

حالياً هذه الأزرار غير موجودة داخل لوحة المهمة.

## 6. مكونات شاشة القسم المالي الحالية

كل قسم مالي يحتوي على نفس البنية:

### 6.1 رأس الصفحة

يعرض:

- عنوان القسم.
- وصف القسم.
- زر الإجراء الرئيسي للمهمة الحالية.

### 6.2 شبكة المؤشرات

تعرض 4 مؤشرات تقريباً لكل قسم.

المؤشرات تساعد المستخدم على فهم حالة القسم بسرعة.

أمثلة:

- قيود اليوم.
- ميزان المراجعة.
- فواتير مفتوحة.
- إجمالي مستحق.
- رصيد البنوك.
- ضريبة مستحقة.
- تكلفة إنتاج.

### 6.3 بطاقات المهام

تعرض المهام أو الوحدات داخل القسم.

كل بطاقة تحتوي:

- أيقونة.
- عنوان المهمة.
- وصف مختصر.

### 6.4 لوحة تفاصيل المهمة

تعرض:

- عنوان المهمة.
- وصف المهمة.
- Workflow من عدة خطوات.
- جدول بيانات.
- نسخة موبايل للصفوف.

### 6.5 جدول بيانات المهمة

الأعمدة الحالية:

| العمود | وظيفته |
|---|---|
| المرجع | كود الحركة أو البند |
| البند | اسم الحساب أو العملية أو المستند |
| المسؤول | الشخص أو القسم المسؤول |
| القيمة | مبلغ أو نسبة أو قيمة مالية |
| التاريخ | تاريخ الحركة |
| الحالة | حالة الصف |

## 7. الأقسام الفرعية الموجودة في المالية

قسم المالية يحتوي حالياً 8 أقسام فرعية:

| القسم | المسار | طبيعة القسم |
|---|---|---|
| المحاسبة العامة | `/finance/general-ledger` | الدليل، القيود، الميزان، القوائم |
| الحسابات الدائنة | `/finance/payable` | الموردون، الفواتير، أوامر الدفع |
| الحسابات المدينة | `/finance/receivable` | العملاء، التحصيل، أعمار الديون |
| الخزينة والنقدية | `/finance/treasury` | بنوك، صناديق، تدفقات، مطابقة |
| الأصول الثابتة | `/finance/fixed-assets` | أصول، إهلاك، صيانة، استبعاد |
| الموازنة والتخطيط | `/finance/budgeting` | موازنات، انحرافات، تحليل |
| الضرائب والامتثال | `/finance/tax` | احتساب، إقرارات، امتثال |
| التكاليف | `/finance/cost` | إنتاج، ربحية، COGS |

## 8. تفاصيل قسم المحاسبة العامة

المسار:

```text
/finance/general-ledger
```

طبيعة القسم:

هذا القسم هو أساس المحاسبة المالية. يفترض أن يحتوي على شجرة الحسابات، القيود، ميزان المراجعة، والقوائم المالية.

المؤشرات الحالية:

- قيود اليوم.
- ميزان المراجعة.
- حسابات نشطة.
- قيود تحتاج مراجعة.

مسار العمل الحالي:

1. إنشاء القيد.
2. مراجعة التوازن.
3. اعتماد المحاسب.
4. الترحيل إلى الأستاذ.

المهام:

| المهمة | زر الإجراء الرئيسي | وظيفة المهمة |
|---|---|---|
| دفتر الأستاذ العام | فتح شجرة الحسابات | عرض الحسابات والأرصدة والحركات |
| قيود اليومية | قيد يومية جديد | تسجيل ومراجعة القيود |
| ميزان المراجعة | تحديث الميزان | فحص توازن المدين والدائن |
| القوائم المالية | تجهيز القوائم | تحضير الميزانية والدخل والتدفقات |

ما يجب أن يحتويه مستقبلاً:

- شاشة مستقلة لشجرة الحسابات.
- نموذج قيد يومية.
- اعتماد وترحيل.
- منع الترحيل إذا القيد غير متوازن.
- أرصدة افتتاحية وختامية.
- إقفال فترة.
- تصدير ميزان مراجعة.

## 9. تفاصيل قسم الحسابات الدائنة

المسار:

```text
/finance/payable
```

طبيعة القسم:

يدير التزامات الشركة تجاه الموردين والمدفوعات المستحقة.

المؤشرات الحالية:

- فواتير مفتوحة.
- إجمالي مستحق.
- أوامر دفع.
- مطابقات شراء.

مسار العمل الحالي:

1. استلام فاتورة المورد.
2. مطابقة الشراء.
3. اعتماد أمر الدفع.
4. تنفيذ السداد.

المهام:

| المهمة | زر الإجراء الرئيسي | وظيفة المهمة |
|---|---|---|
| فواتير الموردين | فاتورة مورد جديدة | متابعة فواتير الشراء والاستحقاق |
| أوامر الدفع | إنشاء أمر دفع | إصدار واعتماد مدفوعات الموردين |
| مطابقة أوامر الشراء | بدء مطابقة | مطابقة الفاتورة مع الشراء والاستلام |
| جدول المدفوعات | عرض الجدول | متابعة الاستحقاقات القادمة |

ما يجب أن يحتويه مستقبلاً:

- سجل فواتير الموردين.
- نموذج فاتورة مورد.
- ربط بأوامر الشراء.
- ربط بالاستلام المخزني.
- اعتماد متعدد المستويات.
- جدول سيولة.
- منع الدفع لفاتورة غير معتمدة.

## 10. تفاصيل قسم الحسابات المدينة

المسار:

```text
/finance/receivable
```

طبيعة القسم:

يدير مستحقات الشركة من العملاء والتحصيلات وأعمار الديون وكشوف الحساب.

المؤشرات الحالية:

- فواتير عملاء مفتوحة.
- مبالغ للتحصيل.
- تحصيلات اليوم.
- مطابقات عملاء.

مسار العمل الحالي:

1. إصدار الفاتورة.
2. متابعة الاستحقاق.
3. تسجيل القبض.
4. مطابقة كشف الحساب.

المهام:

| المهمة | زر الإجراء الرئيسي | وظيفة المهمة |
|---|---|---|
| فواتير العملاء | فتح فواتير العملاء | متابعة الفواتير الآجلة |
| تتبع المدفوعات الواردة | سند قبض جديد | تسجيل التحصيلات |
| إدارة الديون والمتأخرات | عرض أعمار الديون | تحليل التأخير والتحصيل |
| كشوف الحساب | تجهيز كشف حساب | تجهيز كشف عميل للطباعة أو الإرسال |

ما يجب أن يحتويه مستقبلاً:

- ربط بفواتير البيع.
- سندات قبض فعلية.
- أعمار ديون حسب العميل.
- تنبيهات تحصيل.
- مطابقة كشف حساب.
- إرسال كشف حساب PDF أو WhatsApp.

## 11. تفاصيل قسم الخزينة والنقدية

المسار:

```text
/finance/treasury
```

طبيعة القسم:

يدير البنوك والصناديق والتدفقات النقدية والمطابقات.

المؤشرات الحالية:

- رصيد الصناديق.
- رصيد البنوك.
- تسويات مفتوحة.
- تدفق 7 أيام.

مسار العمل الحالي:

1. تسجيل الحركة.
2. ربط الحساب.
3. مطابقة البنك.
4. إقفال يومي.

المهام:

| المهمة | زر الإجراء الرئيسي | وظيفة المهمة |
|---|---|---|
| إدارة البنوك والأرصدة | إدارة الحسابات | عرض وإدارة الحسابات البنكية والصناديق |
| التسويات البنكية | مطابقة جديدة | مطابقة كشف البنك مع النظام |
| التدفقات النقدية | عرض التوقع | توقع الداخل والخارج |
| إدارة الصناديق | إقفال صندوق | إقفال الصناديق اليومية والعهد |

ما يجب أن يحتويه مستقبلاً:

- حسابات بنكية متعددة.
- صناديق نقدية.
- تحويلات بين حسابات.
- مطابقة بنكية تفاعلية.
- كشف حركة بنك.
- إقفال صندوق يومي.
- تقرير تدفق نقدي.

## 12. تفاصيل قسم الأصول الثابتة

المسار:

```text
/finance/fixed-assets
```

طبيعة القسم:

يدير سجل الأصول، الإهلاك، الصيانة، والاستبعاد.

المؤشرات الحالية:

- عدد الأصول.
- القيمة الدفترية.
- إهلاك شهري.
- صيانة قادمة.

مسار العمل الحالي:

1. تسجيل الأصل.
2. تحديد موقع ومسؤول.
3. احتساب الإهلاك.
4. إقفال أو استبعاد.

المهام:

| المهمة | زر الإجراء الرئيسي | وظيفة المهمة |
|---|---|---|
| سجل الأصول | إضافة أصل | تسجيل الأصول ومواقعها ومسؤوليها |
| الإهلاك والإطفاء | احتساب الإهلاك | احتساب إهلاك شهري وقيود الإقفال |
| الصيانة والتقييم | جدولة صيانة | متابعة الصيانة والتقييم |
| بيع أو إتلاف الأصول | استبعاد أصل | تسجيل الاستبعاد والربح أو الخسارة |

ما يجب أن يحتويه مستقبلاً:

- نموذج أصل ثابت.
- نوع الأصل.
- موقع الأصل.
- مسؤول الأصل.
- طريقة الإهلاك.
- قيود إهلاك تلقائية.
- سجل صيانة.
- استبعاد وبيع.

## 13. تفاصيل قسم الموازنة والتخطيط

المسار:

```text
/finance/budgeting
```

طبيعة القسم:

يدير التخطيط المالي، إعداد الموازنات، مقارنة الفعلي بالمخطط، وتحليل الانحرافات.

المؤشرات الحالية:

- موازنة السنة.
- انحراف مصاريف.
- انحراف إيرادات.
- بنود تحتاج قرار.

مسار العمل الحالي:

1. إعداد الخطة.
2. توزيع البنود.
3. مقارنة الفعلي.
4. إجراء تصحيحي.

المهام:

| المهمة | زر الإجراء الرئيسي | وظيفة المهمة |
|---|---|---|
| الميزانيات التقديرية | إعداد موازنة | إعداد موازنة سنوية أو ربع سنوية |
| مقارنة الفعلي بالمخطط | تحليل الانحراف | مقارنة الأداء الفعلي بالمخطط |
| تحليل الانحرافات | تسجيل توصية | تسجيل قرارات وتصحيحات |

ما يجب أن يحتويه مستقبلاً:

- مراكز تكلفة.
- إدخال موازنة.
- اعتماد موازنة.
- مقارنة شهرية.
- رسوم انحراف.
- توصيات إدارية.

## 14. تفاصيل قسم الضرائب والامتثال

/finance/tax

طبيعة القسم:

- ضريبة مخرجات.
- ضريبة مدخلات.

- تنبيهات امتثال.

مسار العمل الحالي:

1. تجميع الحركات.
2. احتساب الضريبة.
3. مراجعة الإقرار.
4. أرشفة المستندات.

المهام:

ما يجب أن يحتويه مستقبلاً:

- فترة ضريبية.

- صافي الضريبة.
- سجل مستندات.

1. تجميع المواد.

- تحليل هامش منخفض.
- توزيع تكاليف غير مباشرة.
- تقرير COGS.

## 16. تقييم تصميم قسم المالية الحالي

### 16.1 نقاط القوة

- بنية الأقسام واضحة.
- كل قسم له مؤشرات خاصة.
- كل قسم له Workflow.
- المهام داخل القسم واضحة كبطاقات.
- تغيير المهمة داخل الصفحة سريع وسلس.
- البيانات معزولة في `financeWorkspace`.
- الواجهة تدعم العربية والإنجليزية من خلال `FinanceText`.
- لا توجد تعديلات مدمرة على تصميم النظام العام.

### 16.2 نقاط الضعف

- المهام لا تفتح شاشات مستقلة.
- زر الإجراء الرئيسي لا ينفذ وظيفة فعلية.
- الجداول موحدة جداً بين كل المهام.
- لا توجد فلاتر.
- لا توجد نماذج إدخال.
- لا يوجد اعتماد أو ترحيل فعلي.
- لا توجد طباعة أو تصدير.
- لا توجد صلاحيات أو حالات workflow حقيقية.
- لا يوجد ربط بالبيانات الحقيقية أو API.
- التجربة أقرب إلى Dashboard/Preview وليست Workspace مالي كامل.

## 17. الفرق بين قسم المالية وقسم التقارير بعد تطوير التقارير

قسم التقارير أصبح يتجه إلى نموذج:

- شاشة مستقلة لكل مهمة.
- فلاتر.
- تطبيق تقرير.
- حالة فارغة قبل التطبيق.
- ملخص بعد التطبيق.
- عرض مختلف حسب نوع التقرير.
- طباعة وPDF وExcel.

أما قسم المالية حالياً لا يزال يعمل بنموذج:

- صفحة واحدة للقسم.
- اختيار مهمة داخل نفس الصفحة.
- لوحة تفاصيل عامة.
- جدول موحد.
- زر إجراء غير منفذ.

هذا ليس خطأ بالضرورة، لأن المالية تشغيلية أكثر من التقارير، لكن الأقسام المالية الكبيرة ستحتاج لاحقاً شاشات مستقلة مثل:

- قيد يومية جديد.
- شاشة فاتورة مورد.
- شاشة سند قبض.
- شاشة مطابقة بنكية.
- شاشة أصل ثابت.
- شاشة احتساب ضريبة.
- شاشة احتساب تكلفة.

## 18. التصميم المقترح لتطوير قسم المالية

- جدول حركة.
- إجراءات مالية حقيقية.

## 20. توصيات قصيرة المدى

   - قيد يومية.

   - أصل ثابت.

5. تخصيص الجداول حسب نوع المهمة.

## 23. الخلاصة

قسم المالية حالياً أصبح منظماً ومجهزاً كواجهة أمامية أولية:

- أقسام فرعية واضحة.
- مؤشرات لكل قسم.
- مهام داخلية.
- Workflow.
- جداول بيانات.
- زر إجراء رئيسي متغير.

لكن القسم لا يزال يحتاج نقلة من **عرض واجهة مالية** إلى **تشغيل مالي حقيقي**.

الخطوة التالية المنطقية ليست إضافة بطاقات أكثر، بل تحويل أهم المهام إلى شاشات مستقلة أو نوافذ عمل فعلية، مع:

- نماذج إدخال.
- اعتماد.
- ترحيل.
- طباعة.
- تصدير.
- فلاتر.
- سجل تدقيق.

بهذا يصبح قسم المالية مختلفاً عن التقارير:

- التقارير: استخراج وتحليل وطباعة.
- المالية: إدخال، اعتماد، ترحيل، ومتابعة حركة مالية.

وهذا الفصل مهم جداً حتى لا تختلط تجربة المستخدم بين "تشغيل مالي" و"قراءة تقرير".
